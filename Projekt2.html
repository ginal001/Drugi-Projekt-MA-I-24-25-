Naszym zadaniem jest napisanie funkcji znajdujcej minimum funkcji. Jednoczenie poszukujemy metod, ktre nie zakadaj rniczkowalnoci naszej funkcji oraz mogce mie wiele minimum lokalnych. W szczeglnoci jako test bierzemy funkcj $$ L^k(x) = x^2(\sin kx + 2). $$ Najpierw dokonamy wizualizacji naszej funkcji.

Jak widzimy nasza funkcja ma wiele minimw lokalnych. Przejdmy do opisu metody, ktrej uyjemy do znalezienia minimum tej funkcji.

Algorytm Symulowanego Wyarzania

Niech $f \colon X \to \mathbb{R}$ bdzie funkcj, ktrej minimum chcemy znale.

1. Inicjalizacja

  • Wybierz pocztkowe rozwizanie $x^{(0)} \in X$ (np. losowo).
  • Ustaw pocztkow temperatur $T_0 > 0$.
  • Zdefiniuj schemat schadzania, np. wykadniczy:
    $$ T_{k+1} = \alpha \, T_k, \quad \alpha \in (0,1). $$

2. Gwna ptla

Powtarzaj dla $k = 0,1,2,\dots$, a do spenienia warunku stopu:

  1. Generacja ssiada
    $$ x' = x^{(k)} + \delta, \quad \delta \sim \mathcal{U}(-\varepsilon,\varepsilon). $$

  2. Ocena rnicy energii
    $$ \Delta = f(x') - f\bigl(x^{(k)}\bigr). $$

  3. Kryterium akceptacji
    $$ x^{(k+1)} = \begin{cases} x', & \text{jeli } \Delta \le 0,\\[4pt] x' & \text{z prawdopodobiestwem } \exp\bigl(-\tfrac{\Delta}{T_k}\bigr),\\[4pt] x^{(k)}, & \text{w przeciwnym razie.} \end{cases} $$

  4. Schadzanie
    $$ T_{k+1} = \alpha \, T_k. $$

  5. Sprawdzenie warunku stopu
    Przykadowe warunki:

    • $k$ osigno maksymaln liczb iteracji,
    • $T_k$ spado poniej zadanej minimalnej wartoci,
    • przez ostatnie $\ell$ iteracji nie byo akceptacji lepszego rozwizania.

3. Zwrcenie wyniku

Po zakoczeniu ptli zwr najlepsze $x$ znalezione w trakcie, tzn.
$$ x^* = \arg\min_{0 \le i \le k} f\bigl(x^{(i)}\bigr). $$

Przejdmy do samego kodu. Zamierzamy szuka minimum, pki nie znajdziemy czego mniejszego ni $10^{-6}$ (minimum to 0). Nastpnie zwrcimy liczb krokw potrzebn do znalezienia tego minimum (jeli zostao ono osignite) lub 50000 jeli to si nie stao. Definiujemy nieco inny schemat schadzania -- domylnie na danej temperaturze wykonujemy 200 krokw (czyli 200 losowa o danym prawdopodobiestwie przejcia). Dodatkowo aktualna temperatura bdzie okrelaa maksymaln odlego ssiada.

In[3]:
def sa_steps(k, init_temp=2.0, n = 50000, alpha=0.85, steps_per_temp=200, threshold=1e-6):
    x = random.uniform(-10, 10)
    best_val = L(k, x) 
    T = init_temp 
    steps = 0  #inicjalizacja
    while n > steps:  #naszym kryterium stopu jest maksymalna liczba krokw lub prg
        for _ in range(steps_per_temp):
            dx = random.uniform(-1, 1) * T
            x_new = min(max(x + dx, -10), 10) #generacja ssiada
            v_new = L(k, x_new)
            steps += 1  # kady krok = jedna ewaluacja nowego x
            if v_new < best_val:
                best_val = v_new
            if v_new < threshold:  # warunek stopu, minimum jest w 0
                return x, steps
            v_curr = L(k, x)
            if v_new < v_curr or random.random() < math.exp((v_curr - v_new) / T):
                x = x_new
        T *= alpha
    
    return x, steps

random.seed(42)
k_vals = [1, 2, 3, 4] + [5*k for k in range(1, 21)]
trials = 20

# Obliczanie redniej liczby krokw dla kadego k
avg_steps = []
for k in k_vals:
    mean = 0
    steps_list = [sa_steps(k)[1] for _ in range(trials)]
    for value in steps_list:
        mean += value
    mean = mean / trials
    avg_steps.append(mean)

Ostatecznie, moemy odnotowa zbieno naszej metody dla $K > 3$ w okoo staej liczbie krokw. Warto jeszcze zestawi nasz metod z naiwn metod Monte Carlo.

def monte_carlo(k, n = 50000, threshold = 1e-6):
    steps = 0
    for _ in range(n):
        steps += 1
        x = random.uniform(-10, 10)
        if L(k,x) < threshold:
            return x, steps
    return x, steps
